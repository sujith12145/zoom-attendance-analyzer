Core Features

Everything you need for attendance management

From raw Zoom CSVs to beautiful reports and ML predictions — fully automated.

🔍

Intelligent Name Matching

Roll-number priority matching, RapidFuzz token-sort ratio, and Levenshtein distance ensure accurate student identification regardless of name format, case, or typos.

⏱️

Multi-Session Merge

Students who leave and rejoin are automatically detected. All sessions are merged and total duration is summed before marking Present or Absent.

🎛️

Dynamic Thresholds

Adjust the minimum required duration (e.g., 45 minutes) and risk thresholds directly from the UI sidebar — no code changes needed.

📊

Rich Analytics Dashboard

Interactive Plotly charts: pie, bar, histogram, heatmap, gauge, monthly trend lines, top/low performers, and risk breakdowns.

🤖

ML Risk Predictions

Random Forest classifies each student as Green/Yellow/Red. Gradient Boosting forecasts future class attendance %. Isolation Forest detects sudden drops.

📄

Multi-format Reports

Download attendance reports as CSV, multi-sheet colour-coded Excel workbooks, or formatted PDF documents with summaries and tables.

🗄️

Persistent SQLite Storage

All data is stored in a local SQLite database. Historical attendance is maintained across sessions with full CRUD operations.

🧹

Smart Data Cleaning

Automatically handles encoding issues, extra spaces, special characters, duplicate entries, inconsistent capitalization, and metadata headers in Zoom exports.

Workflow

How It Works

Three simple steps from upload to insights.

1

Upload Master Student List

Upload a CSV or Excel file with student roll numbers and names. The system normalises names, removes duplicates, and stores the roster permanently. Supported column variants are auto-detected.

2

Upload Zoom Participant Report

After each Zoom class, export the participant report and upload it. The system automatically detects columns, cleans names, extracts roll numbers from display names, merges multi-session entries, and matches each participant to the master list using a 4-tier matching pipeline.

3

View Analytics, ML Predictions & Reports

Explore the dashboard for attendance trends, risk levels, and student-wise analytics. Run ML models to get next-class attendance predictions and at-risk student alerts. Download reports in CSV, Excel, or PDF.

Machine Learning

AI-Powered Risk Intelligence

Three complementary ML models working together to predict, forecast, and detect anomalies.

🌲

Random Forest Classifier

Classifies each student into Green / Yellow / Red risk based on 9 engineered features: rolling attendance average, streak counts, trend slope, avg duration, and more.

📈

Gradient Boosting Regressor

Auto-regressive forecasting of class-level attendance percentages. Predicts the next 3 upcoming sessions using rolling window features and trend slope.

🔎

Isolation Forest

Unsupervised anomaly detection identifies students whose recent attendance deviates sharply from their own baseline — catching sudden attendance drops early.

⚙️

Feature Engineering

9 features: overall attendance %, last-N rolling average, trend slope, consecutive streaks (present & absent), average/std duration, day-of-week pattern.

Risk Classification

Three-Tier Risk System

Configurable thresholds let you define what "safe", "at-risk", and "critical" means for your course.

🟢 GREEN
≥ 75%

Safe

Student is attending regularly. No immediate action required. Keep up the good work!

🟡 YELLOW
50–74%

At Risk

Attendance is slipping. Send a reminder or schedule a check-in with the student.

🔴 RED
< 50%

Critical

Student may fail the course. Immediate intervention — contact student and/or guardian.

All thresholds are adjustable per-session via the UI sidebar or permanently in config.py.

Quick Start

Up and running in 3 commands

Works on Windows, macOS, and Linux with Python 3.11+

 1. Clone & Navigate

git clone https://github.com/your-repo/zoom-analyzer.git
cd zoom-analyzer/zoom_analyzer

 2. Install Dependencies

# Linux / macOS
chmod +x setup.sh && ./setup.sh

# Windows
setup.bat

# Or manually:
pip install -r requirements.txt

 3. Launch Application

streamlit run app.py

# App opens at:
http://localhost:8501
