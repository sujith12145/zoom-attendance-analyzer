## 🚀 Quick Start

### Prerequisites
- Python 3.11+ 
- pip

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/your-repo/zoom-attendance-analyzer.git
cd zoom-attendance-analyzer/zoom_analyzer

# 2. (Recommended) Create virtual environment
python -m venv venv
source venv/bin/activate          # Linux/macOS
venv\Scripts\activate.bat         # Windows

# 3. Install dependencies
pip install -r requirements.txt

# 4. Generate sample data (optional but recommended for testing)
python generate_sample_data.py

# 5. Launch the application
streamlit run app.py
```

Or use the one-click setup scripts:
```bash
chmod +x setup.sh && ./setup.sh   # Linux/macOS
setup.bat                         # Windows
```

The app will open at **http://localhost:8501**

## 🔧 Configuration (`config.py`)

All tuneable parameters live in `config.py` — no code changes required elsewhere:

```python
# Attendance threshold (minutes to be marked Present)
DEFAULT_MIN_DURATION_MINUTES = 45

# Risk classification thresholds
DEFAULT_GREEN_THRESHOLD  = 75    # % → Green (Safe)
DEFAULT_YELLOW_THRESHOLD = 50    # % → Yellow (At-Risk)
# Below 50% → Red (Critical)

# Fuzzy matching
FUZZY_SCORE_CUTOFF = 80          # 0-100; higher = stricter matching

# ML settings
ML_MIN_SESSIONS_FOR_PREDICTION = 3
ML_LOOKBACK_WINDOW             = 5
```

All thresholds can also be adjusted dynamically from the UI sidebar without restarting.

## 🔍 Name Matching Pipeline

The system uses a **4-tier cascading match** for each Zoom participant:

```
1. 🔢 Roll Number (Exact)  ← Highest priority
   Regex extracts roll patterns like "CS-21-045" from display names

2. ✅ Name (Exact, normalised)
   Both names lowercased, special chars cleaned, compared directly

3. 🔍 RapidFuzz Token-Sort Ratio
   Handles word reordering ("Smith John" ≈ "John Smith")
   Default cutoff: 80/100

4. 📏 Levenshtein Distance Fallback
   Character-level edit distance; score = (1 - dist/maxlen) × 100
```

## 📊 Data Models

### SQLite Tables

**`master_students`**
```sql
roll_number TEXT PRIMARY KEY
name        TEXT NOT NULL
name_clean  TEXT            -- normalised for matching
```

**`classes`**
```sql
class_id    INTEGER PRIMARY KEY AUTOINCREMENT
class_name  TEXT
class_date  TEXT            -- ISO: YYYY-MM-DD
subject     TEXT
notes       TEXT
created_at  TEXT
```

**`attendance`**
```sql
id               INTEGER PRIMARY KEY AUTOINCREMENT
class_id         INTEGER     -- FK → classes
roll_number      TEXT
name             TEXT
total_minutes    REAL
sessions         INTEGER     -- number of join/leave cycles
status           TEXT        -- Present | Absent | Unmatched
match_method     TEXT        -- roll_exact | name_exact | fuzzy_rapidfuzz | levenshtein | absent
match_score      REAL        -- 0-100
min_duration_used REAL
```

## 🤖 Machine Learning Module

### Models

| Model | Algorithm | Task |
|-------|-----------|------|
| `AttendanceRiskPredictor` | `RandomForestClassifier` | Classify student risk: Green/Yellow/Red |
| `AttendanceTrendForecaster` | `GradientBoostingRegressor` | Forecast next-class attendance % |
| Anomaly Detection | `IsolationForest` | Detect sudden attendance drops |

### Features (9 total)
| Feature | Description |
|---------|-------------|
| `total_classes` | Number of sessions student has attended |
| `attendance_pct` | Overall historical attendance percentage |
| `last_n_present` | Rolling average of last N classes |
| `last_n_trend` | Linear slope of recent attendance (improving/declining) |
| `streak_present` | Consecutive present count (current streak) |
| `streak_absent` | Consecutive absent count (current streak) |
| `avg_duration` | Average minutes spent in class |
